## üí° Key Learnings

1. **Server Components Benefits:**
   - Direct Prisma queries without API layer
   - Better performance with SSR
   - Simpler code structure
   - Automatic data fetching on server

2. **Database Seeding Best Practices:**
   - Use upsert to prevent duplicates
   - Export/import via JSON for version control
   - Separate export script from seed script
   - Include all related data for complete restore

3. **Component Props Consistency:**
   - Always check component interfaces
   - Use object props for complex data
   - Maintain consistent prop naming
   - Document expected prop structure

4. **Button Consistency:**
   - Standardize padding across all buttons
   - Use utility classes for consistency
   - Test on multiple screen sizes
   - Document design system tokens

5. **Next.js 15 Patterns:**
   - Server Components by default
   - Client Components only when needed
   - Params as Promise in dynamic routes
   - Proper error boundaries with notFound()
