## üí° Technical Decisions Made

### 1. Dedicated Component Folders

**Decision:** Create `/components/about/` and `/components/blog/` folders  
**Reason:** Better organization, easier to maintain, clear separation of concerns

### 2. Article Data in `/lib/`

**Decision:** Store article data in `src/lib/articles.ts`  
**Reason:** Centralized data management, easy to migrate to CMS later

### 3. Dynamic Routing for Articles

**Decision:** Use `[slug]` dynamic routing  
**Reason:** SEO-friendly URLs, scalable for many articles

### 4. Static Generation

**Decision:** Use `generateStaticParams()` for articles  
**Reason:** Better performance, faster page loads, good for SEO

### 5. Client Component for Article Content

**Decision:** Mark `article-content.tsx` as `'use client'`  
**Reason:** Needed for styled-jsx, doesn't affect SEO (content still SSR)
